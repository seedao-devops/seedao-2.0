# SeeDAO 2.0 — 用户旅程 (User Journeys)

本文档定义了 MVP 四大意图模块的用户旅程，用于填补产品规格表中空白的「User Journey」栏位。

---

## 1. 吸引目标用户 (Attract Wanted Users)

### 旅程 A — 新用户注册

1. 访客打开网站首页，点击「注册」按钮
2. 填写注册表单：邮箱、手机号（+86）、密码、确认密码
3. 系统校验：邮箱格式及唯一性、手机号 E.164 格式及唯一性、密码强度
4. 提交后系统发送双通道验证：邮箱收到验证链接、手机收到短信验证码
5. 用户完成邮箱点击确认 + 输入短信验证码
6. 验证成功后进入待审核状态，页面提示「您的申请已提交，请等待管理员审核」
7. 审核通过 / 拒绝后，用户收到邮件 + 短信通知结果

### 旅程 B — 管理员审核申请

1. 管理员登录后台，进入「审核管理」页面
2. 查看申请列表（分页，默认按提交时间倒序）
3. 通过状态筛选器（待审核 / 已通过 / 已拒绝）和日期范围缩小范围
4. 点击某条申请，查看用户详细信息
5. 选择操作：
   - **通过**：选择内部等级（必填）、填写备注（选填，≤500字）→ 确认提交
   - **拒绝**：填写拒绝原因（必填，≥10字）→ 确认提交
6. 系统自动记录操作日志（操作人、旧状态、新状态、时间、IP）
7. 系统自动向用户发送审核结果通知（拒绝通知包含完整拒绝原因）

## 2. 管理自我成长 (Manage Self-Growth)

### 旅程 A — 个人资料管理

1. 用户登录后进入「个人资料」页面
2. 查看当前头像（DiceBear 随机生成）、昵称、涂鸦横幅
3. 点击头像旁「换一个」按钮重新生成随机头像（每30天限3次）
4. 编辑昵称（3-30字符，支持字母数字下划线连字符句点，大小写不敏感去重，每30天限改3次）
5. 上传或更换涂鸦横幅图片
6. 修改密码：输入当前密码 + 新密码，系统校验后更新

### 旅程 B — 技能管理与成长追踪

1. 用户进入「我的技能」页面，查看已创建的技能列表
2. 点击「添加技能」：填写技能名称（≤50字）、描述（≤500字）、等级（入门/中级/高级/专家）
3. 进入某个技能详情页，查看成长数据面板：总时长、连续天数、最近4周每周时长图表
4. 点击「记录成长」：填写日期、投入时长、地点、笔记（选填）、证据链接（选填）
5. 成长记录自动累加至技能总时长和统计数据
6. 可进行自测评估和申请公开验证

### 旅程 C — 学习愿望清单

1. 用户进入「学习愿望」页面
2. 点击「添加愿望」：填写想学技能名称、目标等级、优先级（高/中/低）
3. 查看愿望列表，通过开关控制每项是否公开（公开后空间主理人可见）
4. 管理已有愿望（编辑、删除）

### 旅程 D — 共学关系

1. 用户进入「共学关系」页面
2. 查看通过教学活动自动形成的关系列表
3. 关系按类型分组展示：学习伙伴（双向）、师生（单向）、导师学员（单向）
4. 每条关系显示：对方昵称、关联技能、最近互动时间

## 3. 通过技能建立连接 (Build Connections Through Skills)

### 旅程 A — 管理员创建与管理共学活动

1. 管理员进入「共学管理」页面，查看活动列表（含草稿/已发布/已归档标签页）
2. 点击「创建活动」，填写必填字段：
   - 标题（≤100字）、描述（≤2000字）、技能类别、主持人姓名
   - 最大参与人数（≥1）、计划开始时间、活动时长（分钟）
3. 保存后默认状态为「草稿」，不对外展示
4. 编辑草稿内容，确认无误后点击「发布」
5. 系统校验计划时间必须在未来 → 发布成功，活动对用户可见
6. 活动进行中可编辑信息（发布状态不变），但不可将最大人数改为低于已报名人数
7. 活动结束后管理员选择「归档」（软删除，保留在管理后台可筛选查看）
8. 仅草稿且无报名者的活动可「硬删除」

### 旅程 B — 重复检测

1. 创建新活动时，若标题和开始时间与已有活动在2小时内匹配
2. API 返回警告头 `X-Warning: POSSIBLE_DUPLICATE`，但仍允许创建
3. 前端展示提醒信息，管理员可选择继续或取消

## 4. 分享经验 (Share Experience)

### 旅程 A — 管理员创建与管理手册

1. 管理员进入「手册管理」页面，查看手册列表
2. 点击「创建手册」，填写：
   - 标题（≤120字）、slug（从标题自动生成，可编辑，URL安全，唯一）
   - 摘要（≤300字，选填）、内容（Markdown，最大1MB）、封面图（选填，≤5MB）
3. 保存后默认「草稿」状态，可预览但不对外展示
4. 点击「发布」将状态改为已发布，公开可见
5. 编辑已发布手册时，系统自动创建版本记录（保留历史内容）
6. 修改 slug 时，旧 slug 存入重定向表，访问旧地址自动 301 跳转到新地址
7. 删除手册时需填写说明信息，用户访问已删除手册看到该说明

### 旅程 B — 公开用户浏览手册

1. 用户进入「手册」页面，浏览已发布手册列表（卡片式，按发布时间倒序分页）
2. 使用搜索框按关键词检索手册标题和内容
3. 点击手册卡片进入详情页，查看完整内容、封面图、发布/更新时间
4. 点击「分享」按钮，弹出分享对话框：
   - 显示手册标题、简短描述、指向手册页面的二维码
   - 适配微信、小红书等平台的社交卡片尺寸
5. 访问不存在或已删除的手册 slug 显示相应提示信息
